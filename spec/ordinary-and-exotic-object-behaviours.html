Built-in Exotic Object Internal Methods and Slots

Record Exotic Objects

A Record object is an exotic object that encapsulates a Record value and exposes virtual string-indexed data properties corresponding to the individual keys set on the underlying Record value. All keys properties are non-writable and non-configurable.

An object is a Record exotic object (or simply, a Record object) if its following internal methods use the following implementations and is an Immutable Prototype Exotic Object. These methods are installed in RecordCreate.

Record exotic objects have the same internal slots as ordinary objects. They also have a [[RecordData]] internal slot.

a Record exotic object _R_

1. If Type(_P_) is a Symbol, return *false*. 1. Let _rec_ be _R_.[[RecordData]]. 1. Assert Type(_rec_) is a Record. 1. For each element _entry_ of _rec_.[[Fields]], do 1. Let _key_ be _entry_.[[Key]]. 1. If ! SameValue(_key_, _P_) is *true*, return *true*. 1. Return *false*.

1. Assert: _R_ is an Object that has [[RecordData]] internal slot. 1. Assert: Type(_P_) is String. 1. Let _rec_ be _R_.[[RecordData]]. 1. Assert Type(_rec_) is Record. 1. If _key_ is _P_, return _val_. 1. Return ~empty~.

Tuple Exotic Objects

A Tuple object is an exotic object that encapsulates a Tuple value and exposes virtual integer-indexed data properties corresponding to the individual entries set on the underlying Tuple value. Tuple exotic objects always have a data property named *"length"* whose value is the number of entries in the underlying Tuple value. All keys properties are non-writable and non-configurable.

An object is a Tuple exotic object (or simply, a Tuple object) if its following internal methods use the following implementations and is an Immutable Prototype Exotic Object.

Tuple exotic objects have the same internal slots as ordinary objects. They also have a [[TupleData]] internal slot.

[[IsExtensible]] ( )

When the [[IsExtensible]] internal method of a Tuple exotic object _T_ is called, the following steps are taken:

1. Return *false*.

If _numericIndex_ is *undefined*, return *undefined*. 1. Let _value_ be ? TupleGet(_T_, _numericIndex_). 1. If _value_ is ~empty~, return *undefined*. 1. Return the PropertyDescriptor { [[Value]]: _value_, [[Writable]]: *false*, [[Enumerable]]: *true*, [[Configurable]]: *false* }.

If _current_ is ~empty~ return *false*. 1. If IsAccessorDescriptor(_Desc_) is *true*, return *false*. 1. If _Desc_.[[Value]] is present, return ! SameValue(_Desc_.[[Value]], _current_). 1. Return *false*.

for
a Tuple exotic object _T_
1. Assert: IsPropertyKey(_P_) is *true*. 1. If Type(_P_) is not Symbol, then 1. If _P_ is *"length"*, return *true*. 1. Let _numericIndex_ be CanonicalNumericIndexString(_P_). 1. [[Set]] ( _P_, _Receiver_ )

for

1. Assert: IsPropertyKey(_P_) is *true*. 1. Return *false*.

for
a Tuple exotic object _T_
1. Assert: IsPropertyKey(_P_) is *true*. 1. If Type(_P_) is a Symbol, return *true*. 1. If _P_ is *"length"*, return *false*. 1. Let _numericIndex_ be CanonicalNumericIndexString(_P_). 1. [[OwnPropertyKeys]] ( )

for
a Tuple exotic object _T_
1. Let _keys_ be a new empty List. 1. Let _tup_ be _T_.[[TupleData]]. 1. Assert: Type(_tup_) is Tuple. 1. Let _len_ be the length of _tup_.[[Sequence]]. 1. Let _index_ be 0. 1. Repeat, while _index_ < _len_, 1. Add ! ToString(_index_) to _keys_. 1. Set _index_ to _index_ + 1. 1. Add *"length"* to _keys_. 1. Return _keys_.
